## 기존 사용자 흐름

1. `/`에 진입하면 `/rounds`로 이동한다.
2. 라운드 목록을 조회하고 첫 번째 라운드를 선택한다.
3. 이전·다음 버튼 또는 선택 상자로 라운드를 변경한다.
4. 경기 행을 선택하면 `/matches/:matchId`로 이동한다.
5. 상세 화면에서 승률, 득점 확률, 팀 통계를 확인한다.
6. 재생·정지·초기화 또는 슬라이더로 경기 시간을 변경한다.

## 후속 확인 항목

- 기록 시점에 `localhost:8080` 백엔드가 실행 중이지 않아 실제 API 응답 예시는 수집하지 못했다.
- 실제 응답 예시는 백엔드 실행 후 원문 그대로 추가한다.
- 브라우저 자동화 연결 오류로 변경 전 화면 캡처는 수집하지 못했다.
- 다음 화면 작업 전 라운드 목록과 경기 상세를 데스크톱·모바일 크기로 확인한다.

## 라운드별 경기 목록 리디자인

### 사용자 관점 변화

- 시즌, 페이지 목적, 현재 라운드를 한 화면에서 확인할 수 있다.
- 이전·다음 버튼과 라운드 선택 상자가 하나의 컨트롤로 통합되었다.
- 경기 목록이 날짜별로 구분되고 각 경기가 하나의 카드로 표시된다.
- 카드 전체를 마우스 또는 키보드로 선택해 상세 화면으로 이동할 수 있다.
- 로딩, 조회 오류, 경기 없음 상태가 서로 다른 화면으로 표시된다.
- 작은 화면에서는 팀 로고·팀명·스코어가 3열 카드로 배치된다.

- 팀 로고를 불러오지 못하면 팀명 첫 글자를 대신 표시한다.

## 타임라인 컨트롤과 실제 경기 시간

### 사용자 관점 변화

- 경기 데이터에 존재하는 실제 최대 분까지 슬라이더로 이동할 수 있다.
- 90분을 넘은 시간은 `90+N'` 형식으로 표시된다.
- 현재 시간이 컨트롤 오른쪽에 큰 숫자로 표시된다.
- 재생, 정지, 초기화 버튼의 활성 상태가 현재 시간과 일치한다.
- 마지막 분에 도착하면 재생이 자동으로 정지한다.
- 마지막 분에서 재생을 누르면 0분부터 다시 시작한다.
- 모바일에서는 재생과 초기화 버튼이 동일한 너비로 배치된다.

### 데이터 표시 정책

- 최대 시간은 승률 및 득점 확률 타임라인 전체에서 가장 큰 `minute` 값으로 계산한다.
- 데이터가 없는 분은 임의로 보간하지 않고 빈 상태를 유지한다.
- 희소 데이터의 의미가 확인되면 별도의 정규화 단계에서 정책을 적용한다.

- 현재 선택한 시간이 세로 점선과 확률별 점으로 강조된다.
- 그래프의 특정 시간을 클릭하면 슬라이더와 현재 확률이 함께 변경된다.
- 그래프에 포커스한 뒤 방향키, Home, End 키로 시간을 이동할 수 있다.
- 모바일에서는 그래프 높이와 축 라벨이 단순화된다.
- 그래프 아래에 현재 시간의 확률이 스크린리더가 읽을 수 있는 텍스트로 제공된다.
