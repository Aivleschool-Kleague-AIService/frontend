## 기존 사용자 흐름

1. `/`에 진입하면 `/rounds`로 이동한다.
2. 라운드 목록을 조회하고 첫 번째 라운드를 선택한다.
3. 이전·다음 버튼 또는 선택 상자로 라운드를 변경한다.
4. 경기 행을 선택하면 `/matches/:matchId`로 이동한다.
5. 상세 화면에서 승률과 득점 확률을 확인한다.
6. 재생·정지·초기화 또는 슬라이더로 경기 시간을 변경한다.

## 타임라인 컨트롤과 실제 경기 시간

### 사용자 관점 변화

- 경기 데이터에 존재하는 실제 최대 분까지 슬라이더로 이동할 수 있다.
- 90분을 넘은 시간은 `90+N'` 형식으로 표시된다.
- 현재 시간이 컨트롤 오른쪽에 큰 숫자로 표시된다.
- 재생, 정지, 초기화 버튼의 활성 상태가 현재 시간과 일치한다.
- 마지막 분에 도착하면 재생이 자동으로 정지한다.
- 마지막 분에서 재생을 누르면 0분부터 다시 시작한다.
- 모바일에서는 재생과 초기화 버튼이 동일한 너비로 배치된다.

### 데이터 표시 정책

- 최대 시간은 승률 및 득점 확률 타임라인 전체에서 가장 큰 `minute` 값으로 계산한다.
- 데이터가 없는 분은 임의로 보간하지 않고 빈 상태를 유지한다.
- 희소 데이터의 의미가 확인되면 별도의 정규화 단계에서 정책을 적용한다.

## 상세 API와 상태 로직 분리

### 구조 변화

- Axios 인스턴스를 `api/client.js` 한 곳으로 통합했다.
- API 기준 주소를 `VITE_API_BASE_URL` 환경변수로 이동했다.
- 경기 기본 정보·득점 확률·승률 API를 `useMatchDetail`에서 병렬 호출한다.
- 요청 취소를 적용해 화면 이탈 후 상태가 갱신되지 않도록 했다.
- API 응답 필드 변환을 경기 정보·득점·승률 Adapter로 분리했다.
- 상세 페이지는 API 필드명 대신 정규화된 데이터만 사용한다.

### 사용자 관점 변화

- 상세 API를 기다리는 동안 명확한 로딩 상태가 표시된다.
- 하나 이상의 상세 요청이 실패하면 오류 메시지와 다시 시도 버튼이 표시된다.
- 분석 데이터가 전혀 없는 경기에는 별도의 빈 상태가 표시된다.
- 다시 시도 시 페이지를 새로고침하지 않고 상세 API를 재요청한다.

## 상세 URL 직접 접근과 새로고침 복구

### 사용자 관점 변화

- `/matches/:matchId` 주소를 새 탭에서 열어도 경기 정보와 확률 데이터를 조회한다.
- 상세 화면을 새로고침해도 팀, 점수, 라운드, 타임라인이 복구된다.
- 목록에서 전달된 경기 정보는 초기 헤더 표시에만 사용하고 서버 응답으로 다시 확정한다.
- 존재하지 않거나 조회할 수 없는 경기에는 오류 화면과 다시 시도 버튼이 표시된다.
- 팀 비교 분석 영역은 화면과 데이터 요청에서 제거되었다.

### 구조 변화

- `GET /matches/:matchId`를 상세 기본 정보 API로 연결했다.
- 목록 응답의 `matchId`와 상세 응답의 `matchid`를 Adapter에서 함께 처리한다.
- 팀 비교 컴포넌트, Mock 데이터, Adapter 및 `team-summary` 요청을 삭제했다.

## 접근성·반응형·코드 정리

### 접근성

- HTML 문서 언어를 한국어로 설정했다.
- 페이지 제목과 서비스 설명 메타데이터를 추가했다.
- 아이콘 전용 버튼에는 접근 가능한 이름을 제공한다.
- 팀 로고는 인접한 팀명과 중복 낭독되지 않도록 장식 이미지로 처리한다.
- 홈·무승부·원정은 색상과 텍스트 라벨을 함께 사용한다.
- 키보드 포커스와 reduced-motion 전역 정책을 유지한다.

### 반응형

- 서비스명은 축소되지 않고 팀 로고 영역만 남은 너비를 사용한다.
- 팀 로고는 동일한 40px 프레임 안에 비율을 유지해 표시한다.
- 767px 이하에서는 헤더 팀 로고를 숨겨 서비스명과 콘텐츠 너비를 확보한다.
- 라운드 카드, 경기 헤더, 타임라인, 그래프에 모바일 레이아웃을 적용했다.

### 정리한 항목

- 사용하지 않는 `MatchList`, `RoundHeader` 컴포넌트
- 사용하지 않는 확률 Mock 데이터
- React 및 Vite 예제 SVG와 Vite 파비콘 설정
- 남아 있던 인라인 스타일과 예제 주석
